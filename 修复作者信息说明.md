## 解决方案

### 方案1：修改Git配置（只影响未来提交）

```bash
# 修改全局配置
git config --global user.name "MRYGP"
git config --global user.email "mrygp@example.com"

# 或者只修改当前仓库的配置
git config user.name "MRYGP"
git config user.email "mrygp@example.com"
```

### 方案2：修改最新提交的作者信息（推荐）

如果需要修改已经推送的提交的作者信息：

```bash
# 修改最新提交的作者
git commit --amend --author="MRYGP <mrygp@example.com>" --no-edit

# 强制推送到远程
git push --force-with-lease origin master
```

### 方案3：修改多个提交的作者信息

如果需要修改历史中的多个提交：

```bash
# 使用git filter-branch（谨慎使用）
git filter-branch --env-filter '
if [ "$GIT_COMMITTER_NAME" = "康康康" ]; then
    export GIT_COMMITTER_NAME="MRYGP"
    export GIT_COMMITTER_EMAIL="mrygp@example.com"
fi
if [ "$GIT_AUTHOR_NAME" = "康康康" ]; then
    export GIT_AUTHOR_NAME="MRYGP"
    export GIT_AUTHOR_EMAIL="mrygp@example.com"
fi
' -- --all
```

## 注意事项

⚠️ **修改已推送的提交需要强制推送**，会重写历史，确保没有其他人在使用这些提交。

## 建议

1. **先确认正确的用户名和邮箱**
2. **修改Git配置**，确保未来提交使用正确的作者信息
3. **如果需要修改历史提交**，使用方案2或方案3

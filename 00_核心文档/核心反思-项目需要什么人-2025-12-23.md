# 核心反思：项目真正需要什么人？

**日期**：2025-12-23  
**背景**：在讨论招聘策略时的深度反思  
**结论**：助贷验证员 > 会计实习生 > 研究实习生

---

## 一、核心洞察

### 贷查查凭什么能做好？
```
答案：因为我们自己跑过、踩过坑、知道银行实际怎么审
```

**如果自己没跑过**：
- 诊断不准 → 推荐的产品客户申请不下来
- 方案不实 → 说能贷300万，实际只批100万
- 话术不对 → 不知道银行会问什么、卡什么

**跑过之后**：
- 知道每个产品的"真实门槛"（官网说的和实际审批是两回事）
- 知道每个银行的"隐形条件"（要有存款、要有流水、要有XXX）
- 知道"材料怎么准备才能过"

---

## 二、三种落地方式都需要验证
```
1. 我们/合伙人帮企业做助贷
2. 合作的中介做助贷
3. 客户自己去银行贷款
```

**不管哪种方式，我们都必须自己先跑通**：
- 自己跑过才知道推荐对不对
- 自己跑过才能教合伙人
- 自己跑过才能给中介提供有价值的方案
- 自己跑过才能告诉客户怎么准备材料

---

## 三、角色重新排序

### 原来的想法

| 角色 | 优先级 | 想法 |
|------|--------|------|
| 会计实习生 | P0 | 整理银行产品数据 |
| 研究实习生 | P1 | 用户研究、话术设计 |
| 合伙人运营 | P2 | 招合伙人、带合伙人 |

### 反思后的判断

| 角色 | 优先级 | 原因 |
|------|--------|------|
| **助贷验证员** | **P0** | 不跑就不知道真实情况，产品做不好 |
| 会计实习生 | P1 | 整理数据，但门槛低，自己也能做 |
| 研究实习生 | P3 | 话术自己最懂，研究太早 |
| 合伙人运营 | P2 | 赵红英不来，先放一放 |

---

## 四、助贷验证员定义

### 核心任务

拿真实公司资料，去银行跑审批，验证能不能贷、能贷多少

### 工作内容

| 任务 | 说明 |
|------|------|
| 线上申请 | 在各银行APP/小程序提交申请 |
| 线下递件 | 去银行网点提交材料 |
| 跟进审批 | 打电话问进度、补材料 |
| 记录结果 | 申请了什么→批了多少→卡在哪→用了多久 |

### 产出

**每个产品的"实测报告"**：
- 申请时间
- 提交材料清单
- 审批时间
- 批准额度/利率
- 被问到的问题
- 卡住的点
- 经验总结

**这些实测数据，才是贷查查的核心资产。**

### 画像

| 维度 | 要求 |
|------|------|
| 核心能力 | 勤快、敢沟通、能跑外勤 |
| 专业背景 | 金融/银行/市场/销售优先，但不限 |
| 性格特点 | 外向、不怕打电话、细心记录 |
| 不需要 | 不需要懂会计、不需要懂AI |

## 五、会计实习生重新定位

### 原来的定位
银行产品数据整理 → 核心岗位

### 现在的判断
**这个活其实门槛不高**：
- 随便一个实习生就能做
- 实在不行手工也能做
- 只要对方愿意做、肯学就行

### 可能的调整
1. **降低招聘优先级**：先招助贷验证员
2. **合并角色**：让助贷验证员顺手整理数据
3. **自己先做**：先手工整理核心产品，后面再招人

---

## 六、研究实习生为什么不急？

### 原来的设想
招一个用户研究实习生，做话术设计、用户访谈

### 现在的判断

| 原因 | 说明 |
|------|------|
| **话术你自己最懂** | 你跟客户聊了这么多，话术是你脑子里的，实习生写不出来 |
| **用户研究太早** | 项目还在验证阶段，先跑通再说"研究优化" |
| **研究实习生定位偏"脑"** | 但现在缺的是"手" |

### 结论
研究实习生延后，等项目跑通后再考虑

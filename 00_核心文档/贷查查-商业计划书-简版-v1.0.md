# 贷查查 · 商业计划书（简版）

**版本**：v1.0  
**日期**：2025-12-26  
**定位**：企业贷款能力诊断与服务平台

---

## 一、一句话说清楚

> **贷查查帮中小企业搞清楚：能贷多少、在哪贷、怎么组合最划算。**
> 
> 通过标准化诊断，连接企业主和贷款服务，赚取服务分润。

---

## 二、我们解决什么问题？

### 企业主的痛点

| 痛点 | 现状 | 我们的解决方案 |
|------|------|---------------|
| 不知道能贷多少 | 问中介，各说各话 | 系统诊断，给出额度区间 |
| 不知道哪家银行合适 | 一家家跑，费时费力 | 跨银行对比，一目了然 |
| 不知道怎么组合最优 | 单一产品，没有全局视角 | 组合策略，最优配比 |
| 被中介忽悠 | 信息不透明 | 诊断报告，数据说话 |

### 银行给不了的东西

| 我们能给 | 银行为什么给不了 |
|---------|-----------------|
| 跨银行额度对比 | 银行只推自己的产品 |
| 多产品组合策略 | 银行不会帮你去别家贷 |
| 改造前后对比 | 相当于教客户包装，监管红线 |

## 三、产品是什么？

### 核心功能

```
输入：企业基本信息（15道题，3分钟）
  ↓
处理：匹配20+银行产品，计算额度
  ↓
输出：体检报告
  • 当前可贷额度
  • 跨银行产品对比
  • 组合策略建议（保守/平衡/激进）
  • 体质提升建议
```

### 产品形态

| 版本 | 用户 | 功能 | 目的 |
|------|------|------|------|
| **公开版** | 企业主/任何人 | 基础诊断 + 改造概要 | 引流 |
| **合伙人版** | 我们的合伙人 | 全功能 + 后端对接 | 成交 |
| **贴牌版**（未来） | 外地中介公司 | 白标系统 | 扩张 |

## 四、怎么赚钱？

### 收入来源

| 来源 | 付费方 | 模式 | 优先级 |
|------|--------|------|--------|
| 银行贷款返佣 | 银行/渠道 | 贷款额0.5-1% | P0 |
| **银行渠道返佣** ⭐新 | 银行 | CPS返佣 | P0 |
| H业务分润 | H业务服务方 | 月息分润20-30% | P0 |
| 资质认定返佣 | 认定机构 | 服务费10-20% | P0 |
| 贴牌软件费 | 外地中介 | 月费/年费 | P1（未来） |

### 长期业务的价值

```
H业务/流水服务 = 长期收入

月份1：新增2个客户 → 月分润1万
月份2：新增2个，存量4个 → 月分润2万
月份6：新增2个，存量12个 → 月分润6万
月份12：存量20+个 → 月分润10万+

客户累积，收入复利增长
```

## 五、商业闭环

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   企业主（公开版）              合伙人（专属版）                   │
│        │                            │                          │
│        ↓                            ↓                          │
│   ┌─────────────────────────────────────────┐                  │
│   │           贷查查诊断系统                 │                  │
│   │   基础诊断 + 组合策略 + 改造概要         │                  │
│   └─────────────────────────────────────────┘                  │
│        │                            │                          │
│        │想详情                       │有完整方案                 │
│        ↓                            ↓                          │
│   【联系我们】←─────────────────────┘                          │
│        │                                                       │
│        ↓                                                       │
│   ┌─────────────────────────────────────────┐                  │
│   │           我们的后端服务                 │                  │
│   │   • 银行贷款对接                        │                  │
│   │   • H业务（开票/流水）                  │                  │
│   │   • 资质认定（科小/高新）               │                  │
│   └─────────────────────────────────────────┘                  │
│        │                                                       │
│        ↓                                                       │
│   【成交 → 分润】                                               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**关键设计**：
- 公开版不给改造详情 → 想知道必须联系我们
- 每份报告都有"联系我们"入口 → 客户进入漏斗
- 后端服务商绑定我们 → 不流失到外部

## 六、用户是谁？

| 用户 | 角色 | 我们和他的关系 | 优先级 |
|------|------|---------------|--------|
| **合伙人** | 获客+服务 | 给他工具，他带客户，我们分润 | P0 |
| **企业主** | 最终买单 | 通过诊断引流，通过服务变现 | P0 |
| **后端服务商** | 交付服务 | 我们带客户，他们服务，返佣给我们 | P0 |
| **外地中介**（未来） | 贴牌客户 | 我们卖软件，他们贡献数据 | P1 |

## 七、竞争优势

### 我们有什么？

| 优势 | 说明 |
|------|------|
| **跨银行视角** | 不是单一银行，是全局最优 |
| **组合策略** | 不是单产品最大化，是组合最优化 |
| **改造路径** | 不只说"能不能"，还说"怎么提升" |
| **实时更新** | 银行政策变了，我们第一时间更新 |
| **数据众包**（未来） | 贴牌中介贡献当地数据，越用越准 |
| **银行合作潜力** ⭐新 | 帮银行筛选精准客户，完成小微贷款KPI，可收渠道返佣 |

### 护城河

| 护城河 | 说明 |
|--------|------|
| **银行政策数据** | 持续积累，难以复制 |
| **预测准确率** | 案例越多越准，新进入者没有 |
| **合伙人网络** | 关系建立需要时间 |
| **后端服务商关系** | 合作磨合需要时间 |

## 八、发展阶段

### Phase 1：跑通闭环（0-6个月）

**目标**：验证产品和商业模式

| 任务 | 验收标准 |
|------|---------|
| 产品MVP上线 | 能完成诊断，输出报告 |
| 跑通实测 | 用公司A验证预测准确性 |
| 签约后端 | 至少各1家（银行渠道、H业务、资质认定） |
| 发展合伙人 | 5-10个种子合伙人 |
| 首批成交 | 10单以上 |

### Phase 2：规模化（6-12个月）

**目标**：扩大规模，建立品牌

| 任务 | 验收标准 |
|------|---------|
| 合伙人扩张 | 50+合伙人 |
| 月成交量 | 50单以上 |
| 开始贴牌 | 试点3-5家外地中介 |
| 数据众包 | 建立数据贡献机制 |

### Phase 3：全国扩张（12个月后）

**目标**：成为行业基础设施

| 任务 | 验收标准 |
|------|---------|
| 贴牌规模化 | 50+贴牌中介 |
| 覆盖城市 | 20+城市 |
| 数据优势 | 政策数据最全最准 |

## 九、风险与应对

| 风险 | 应对 |
|------|------|
| 预测不准，失去信任 | 先跑实测，积累案例，用数据说话 |
| 后端服务差，客户投诉 | 严选后端，先小后大，建立淘汰机制 |
| 后端绕过我们 | 合同约束，保持客户粘性 |
| 被抄袭 | 持续更新，数据壁垒，合伙人网络 |
| 合规风险 | 不碰灰色业务，强调"真实业务基础" |

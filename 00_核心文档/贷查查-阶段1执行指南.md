## 🎯 阶段1：你只做三件事

### 一、跑出3个真案例

**挑3家样本企业（至少2家是你自己的）**：

| 类型 | 特征 | 用途 |
|------|------|------|
| 企业A | 体质好，现在就能贷 | 验证"快速匹配"流程 |
| 企业B | 一般，需要优化结构 | 验证"方案B"流程 |
| 企业C | 较差，需要体质改造 | 验证"H业务"流程（可稍后排） |

**每家企业必须做到**：

- [ ] 填完/口述完「贷款体检7连问 + 诊断问卷」
- [ ] 整理一份清晰的票、税、流水、资产、征信记录
- [ ] 你亲自给出A/B/C三套方案的口头结论
- [ ] 沉淀成「标准案例文件」存到 `04_案例库/内部测试/`

**案例文件至少包含**：
1. 企业体质画像（几句话 + 关键数字）
2. 体检结果（A/B/C评级 + 理由）
3. 方案对比（额度/利率/难度/风险）
4. 实际执行了哪条（先记计划，后补结果）

---

### 二、把"怎么说"定死

**你要做的**：

- [ ] 通读《贷查查-产品说明书v0.1》，改成你的口味 → v0.2
- [ ] 通读《贷查查-风控与话术边界v0.1》，刻进脑子里 → v0.2
- [ ] 在真案例里套用这两份文档，发现问题就改

**红线（背下来）**：

| ❌ 绝对不能说 | ✅ 必须这样说 |
|-------------|-------------|
| 包下款 | 预计额度区间XX-XX万 |
| 100%能做 | 符合条件的话通过率较高 |
| 最低利率3% | 利率区间3%-8%，以银行审批为准 |
| 当天放款 | 最快X个工作日 |
| 我代表银行 | 我帮你匹配银行产品 |

---

### 三、带队：给技术员和实习生明确方向

**你的角色**：拍板 + 定规则，不做搬运工

#### 对技术员拍板3件事：

1. **先做一个"能用的粗糙版诊断"**
   - 输入：7连问关键字段 + 企业基本信息
   - 输出：体质评分(A/B/C) + 建议路径 + 3-5个候选产品
   - 不管什么技术栈，**尽快能用就行**

2. **确认产品数据字段**
   - 开个短会，定死15-20个核心字段
   - 先用这些，其他以后再加

3. **用你的3个案例来测试**
   - 系统算一遍，和你人工结果对比
   - 不合理的记成"待优化清单"

#### 对实习生明确3件事：

1. **第1周**：跟你做2家公司的体检记录 + 扫描贷款
2. **第2周起**：
   - 银行资料归档 + 命名规范
   - 每周录入X个产品到《产品总表》
   - 针对样本企业输出「纸面测算 + 小结」
3. **规则**：看不懂的先自己查 + 问银行，再来问你

## 一、先说一下我们在干嘛

简单讲，我现在在做一套 **"企业贷款体检 + 贷款方案设计 + 合作方分工"** 的系统。

* **前端**：用一套问卷和工具（"贷查查"）帮老板做**资金体检**，看清楚：
  * 现在大概能贷多少？
  * 哪几家银行、哪几类产品适合？
  * 利率大概什么区间？

* **中间**：把企业的 **票、税、流水、资产、征信** 等信息，和各家银行的产品政策匹配，给出 **方案 A/B/C**（现在能做、轻度优化、体质改造）。

* **后端**：对接贷款、过桥、典当、做量、体质改造等合作方，我们主要赚的是**后端服务费 / 返佣**，前面的体检和方案，对老板来说是免费的。

你来这边做会计方向的产品实习生，其实就是帮我把这套东西的"底层数据"和"真实案例"搭起来。

## 二、你要做的核心工作（主线）

下面这些是你这份实习的"主线任务"，基本都围绕 **银行产品 + 企业体检 + 系统落地** 展开。

### 1. 资料归档（把原始材料管起来）

* 把各家银行的产品原始资料（PDF / Word / Excel 等），整理到仓库里的目录 `01_银行政策库/01_原始资料/`
* 按 **银行 / 产品** 规范命名、分类归档（比如：`交行-税金快贷-小微一般纳税人-2024.pdf`）
* 维护一份《原始资料目录表》：哪家银行、哪些产品、最新版是哪一个文件

> 这一步就像给"药柜"贴标签，不然后面大家都找不到药。

---

### 2. 产品录入（把死文档变成活数据）

* 按统一字段，把产品逐条录入《产品总表.xlsx》，字段大致包括：
  * 银行、产品名称、产品类型
  * 额度区间、利率区间、期限
  * 票税要求、流水要求、征信要求
  * 担保/抵押要求
  * 所需材料、审批流程要点
  * 风险备注（比如：隐形收费、必须搭售的产品等）
* 看不懂、文档里没写清楚的字段，可以先留空，并在表里记录为"待补充字段"

> 简单讲：你是在把一堆看不完的说明书，整理成一张大家都能读懂的"产品字典"。

---

### 3. 信息核实（和银行/中介把关键口径问清楚）

* 按我们给你的提纲，通过电话/微信联系银行客户经理或中介，补齐关键信息，比如：
  * 额度算法（按税、按流水、按资产？哪个为主？）
  * 零申报的容忍度（能不能有零申报，连续几个月可以接受？）
  * 一般纳税人的硬性要求（必须多久成立？有没有特殊行业例外？）
  * 隐形条件（比如：必须在该行有结算、必须有一定存款等）
* 把这些口径整理清楚，回填到《产品总表》或备注里

> 这一块很关键，你问出来的每一个细节，都是后面诊断更准确的基础。

---

### 4. 纸面测试（用标准客户来"试产品"）

* 我会跟你一起设计几类标准企业画像（比如 A/B/C 体质的典型企业），包含票、税、流水、资产、征信等关键数据
* 你要用这些"标准企业"，对重点产品做纸面判断：
  * 这类企业能不能过这个产品的准入门槛？
  * 大概能批多少额度？
  * 哪里的风险点最大？
* 最后产出一份《产品 × 体质矩阵》：某类体质的企业，适合哪些产品，不适合哪些产品

> 这部分有点像"模拟审批"，帮助我们看清产品和企业之间的真实匹配关系。

---

### 5. 实战记录（把真实落地的案例记下来）

* 对那些最终真的走到 **批款 / 放款** 的项目，你要帮我做"病例记录"：
  * 用了哪个产品 / 哪几家银行
  * 准备了哪些材料
  * 审批时间大概多久
  * 最终额度、利率、期限是多少
  * 中间被问到的关键问题有哪些
* 整理成《产品实战案例》，沉淀到案例库里

> 这些是真金白银的经验，以后不管是给客户解释、还是教合伙人，都会非常有用。

---

### 6. 和技术协同（让系统吃得懂这些数据）

* 跟技术负责人对齐：每一个字段的定义、取值范围，哪些字段必须有，哪些是选填
* 帮忙提供一些"小样本 JSON"和字段解释，让系统知道每一列是什么含义
* 在系统有了初步推荐功能之后，你要做 **sanity check**：
  * 用真实或标准客户的资料跑一遍系统推荐
  * 看推荐出来的产品合理不合理
  * 不合理的地方做记录，方便我们调整算法和规则

---

### 7. 工作节奏（大致的周节奏）

* **每周初**：和我/技术对齐本周要重点推进的产品或银行
* **周内**：围绕"资料归档 → 产品录入 → 信息核实 → 纸面测试 → 实战跟进"这条线推进
* **周末**：写一份《本周产品库进展小结》，主要内容包括：
  * 本周新增/更新了哪些产品
  * 哪些字段仍然缺口，需要下周重点问
  * 做了哪些纸面测试，有什么发现
  * 系统推荐有什么不合理的地方

然后我们一起开一个简短的复盘，调整下一个阶段的重点。

## 五、几个边界提醒（重要）

实习生容易踩的坑，我先说清楚：

| ❌ 不要做 | ✅ 应该做 |
|----------|----------|
| 自己对客户/银行做任何承诺 | 不确定的事先问我 |
| 不懂的问题编答案 | 直接说"我问一下"，然后来问我 |
| 敏感信息（客户名、具体金额）发到群外 | 所有客户信息只在内部沟通 |
| 闷头干，遇到问题不说 | 遇到任何卡住的地方，第一时间说 |

### 数据与保密（硬红线）

银行、贷款、征信这种领域，数据安全是硬红线：

- 所有与企业、老板、金额、征信相关的信息，**只能在内部工具和内部沟通中使用**
- **不允许**将资料、截图、表格转发给无关第三方（包括个人朋友和社交媒体）
- 如有不确定的外联需求，**先问我再动**
- 离职/实习结束后，删除本地留存的敏感资料

**核心原则：不确定的事，先问再动。问问题永远不丢人。**

## 六、你会跟谁一起配合

你不是在角落里"默默录数据"，而是站在一个重要位置上，和其他人紧密配合：

### 团队结构

```
松阳（老板）
    │
    ├── 合伙人运营（赵/待定）── 负责：招合伙人、带合伙人、出单
    │
    ├── 会计实习生（你）────── 负责：产品库、话术、SOP整理
    │
    └── 产品实习生（待定）──── 负责：工具开发、数据支持
```

### 你和其他人的协作

| 协作对象 | 你给TA什么 | TA给你什么 |
|---------|----------|----------|
| **合伙人运营** | 产品知识、话术库、培训材料 | 实战反馈、哪些话术好用/不好用 |
| **产品实习生** | 产品字段定义、数据样本 | 工具支持、自动化建议 |
| **松阳** | 周报、问题反馈、数据进展 | 方向指导、资源协调 |

### 周会机制

**每周五下午30分钟**：
1. 你汇报：本周整理了什么、更新了什么、卡在哪
2. 运营汇报：本周招了多少人、出了多少单、需要什么支持
3. 产品汇报：本周做了什么功能
4. 松阳：解决卡点、调整方向

**你的位置**：你负责说"数据和规则"，运营负责说"人和转化"。

## 七、如果合作顺利

这些是我希望你真正拿走的"长期收获"，而不只是几个月的实习证明。

### 1. 金融产品的"实战眼光"

* 你会系统地接触到各家银行的小微企业产品：税贷、票贷、流贷、房抵贷等等
* 不只是记条件，而是理解：银行为什么这么设计？票、税、流水、资产和征信，各自起什么作用？
* 这会让你从"只会做账的会计"，往"懂融资逻辑的财务人"迈一步

### 2. 数据整理与分析能力

* 产品总表、体质矩阵、案例表，这些都是结构化数据
* 你会大量使用 Excel、筛选、透视表、简单公式等工具
* 长期下来，你会对"如何把一堆乱七八糟的信息整理成一张清楚的表"有自己的方法论

### 3. 真实业务沟通经验

* 你会跟银行客户经理、中介机构有真实的对话，不是学校里演练，而是问真实产品的关键信息
* 你会慢慢习惯：怎么开口问、怎么追问细节、怎么把对方说的话整理成清楚的业务规则

### 4. 项目与产品视角

* 你不只是一个"录入数据的实习生"，而是参与一个从 0 到 1 的项目：
  * 一起搭一个银行产品数据库
  * 一起搭一个"贷款体检系统"
* 这在你以后写简历、或者找产品/风控/数据岗时，都会是很清晰的一段经历

### 5. AI 实战能力

这一点对我来说很重要：**我希望你比我更会用 AI。**

**AI学习路径（参考）**：
| 阶段 | 目标 |
|------|------|
| 第1-2周 | 熟悉Claude/ChatGPT基础用法，能用AI帮自己查资料、整理信息 |
| 第3-4周 | 尝试用AI辅助产品录入、信息整理、写周报 |
| 之后 | 探索更多场景，反向教我怎么用 |

我会指定一些常用的AI工具让你去学、去试。你的任务之一，就是：
* 先自己摸清楚这些工具怎么用在我们的工作场景里
* 然后**反过来教我**，一起把这套工具融进日常工作流

简单说，你会在真实场景里练习："用 AI 帮忙干活，而不是被 AI 牵着走"。

### 6. 自动化工具与效率意识

出于我的个人习惯，我也会鼓励你去接触一些效率工具，比如：
* RPA（机器人流程自动化）
* Zapier（跨平台自动化）
* 以及你自己感兴趣的其他自动化工具

我不会强压给你某一个工具，但如果你有兴趣，我会很支持你：自己去试、试出结果、然后回来跟我说"这个东西有用/没用，可以怎么用在我们项目里"。

## 七、如果合作顺利

这部分我也坦诚说一下：

* **实习期结束**：如果双方都觉得合作顺利，可以继续兼职/远程合作
* **深度参与**：有机会参与更多核心项目，接触更复杂的业务
* **长期可能性**：表现好的话，可以聊长期合作的可能性

当然，这些都建立在"你觉得这事儿有意思、我觉得你靠谱"的基础上，不是画饼。

### 我们怎么判断合作顺利

**评估维度**：
| 维度 | 说明 |
|------|------|
| **任务完成度** | 产品录入、信息核实、周报等主线任务的完成情况 |
| **沟通反馈** | 遇到问题是否及时说、能不能清楚表达 |
| **学习与改进** | 是否在进步、有没有主动学新东西 |
| **合规意识** | 数据保密、边界感 |

**实习结束后**：
- **A档**：优先考虑长期合作，可提供推荐信
- **B档**：按项目结算后友好结束
- **C档**：中途沟通调整，或提前结束

这不是为了给你压力，而是让你知道"努力方向"和"对等关系"。

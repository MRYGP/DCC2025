## 0. 项目一句话

**DCC2025 / 贷查查 = 企业贷款体检与体质改造系统。**

- **前端**：用「贷查查 · 企业贷款体检」做免费入口，帮老板看清现在能贷多少、为什么、有哪些路可以走
- **中台**：用标准化问卷 + 诊断逻辑 + 案例库，给出方案 A/B/C（现在能做 / 轻度优化 / 体质改造）
- **后端**：对接贷款、过桥、典当、做量、H业务等具体执行方，我们只做「诊断 + 建议 + 分诊 + 协调」，收入来自后端返佣/咨询服务费

### 商业模式核心

> **贷查查 ≠ SaaS产品，而是「企业贷款医生 + 合伙人获客平台」**
> 
> - 我们帮后端服务方（银行/助贷/过桥）获客
> - 后端服务方是我们的"隐性投资人"——我们帮他们拉业务，他们给我们分成
> - 合伙人赚得越多，我们才赚得越多
> 
> 详见：`00_核心文档/贷查查-商业模式与系统架构说明v1.0.md`

## 2. 角色分工

### 2.1 业务负责人（常松阳）

**核心职责**：
- 定位与商业模式设计：免费体检 + 方案A/B/C + 后端分诊 + 合伙人模式
- 和真实企业主对话，跑出第一批可落地的内部案例
- 搭建文档体系：产品说明、风控边界、合伙人手册、合作方规则等
- 拍板关键规则：额度预测边界、风险提示、什么不做、谁能当合作方

**短期重点**：
- 用自有公司 + 少量熟人企业，跑出3–5个完整体检案例
- 约定1–2家可信的后端合作方
- 带着技术负责人和实习生，对齐"什么叫一份合格的体检报告"

### 2.2 技术负责人（技术员团队）

**核心职责**：
- 把「7连问 + 产品库 + 案例库」变成一个可调用的诊断服务
- 设计并实现产品数据结构、诊断算法、报告生成逻辑
- 后续接入RAG/Agent，把文档知识用起来

**短期重点**：
- 搭建基础项目骨架（如FastAPI）
- 实现 `/diagnose/basic`：输入7连问 → 输出体质等级 + 方向建议
- 设计并落地产品总表的字段schema + 导入脚本
- 提供一个能自动生成「体检报告草稿」的最小可用版本

### 2.3 会计方向产品实习生

**核心职责**：
- 管好银行产品与企业案例的"原材料"：归档、录入、质检
- 协助做真实企业的体检记录与纸面扫描
- 配合技术测试诊断结果是否合理

**短期重点**：
- 跟随业务负责人，完成自有公司A/B的体检记录与纸面扫描
- 整理银行原始资料 → 命名规范 → 录入产品总表首批10–15个产品
- 按模板整理案例文档，形成可复用的Case Study

### 2.4 合伙人 / 助贷中介（未来）

**定位**：
- 只负责获客 + 体检 + 反馈体检结果，不负责方案设计与资金落地
- 不代表任何银行或资金方做承诺

## 3. 分阶段路线图（0 → 1 → N）

### 阶段1：创始人自用期（当前）

**目标**：
- 用自有公司 + 熟人企业，跑通"体检 → 报告 → 方案 → 后端落地"全流程
- 积累3–10个真实的before/after案例

**关键动作**：
- 完成本周《近期执行计划》中列出的6个硬任务
- 每跑完一个案例，立即整理成 `04_案例库/内部测试/案例-公司X-体检与方案v0.x.md`
- 在实践中调整问卷、诊断逻辑、报告模板

### 阶段2：内圈合伙人试点（3–5人）

**前提**：
- 有了可用的诊断工具（basic版）
- 有了若干真实案例 + 体检报告模板
- 有明确的风控边界与话术规范

**关键动作**：
- 精选3–5个熟悉的代账/保险/工程/贸易朋友，作为内圈合伙人
- 提供：简化版问卷、小工具入口、体检报告模板、基础分成规则
- 要求每人1个月内完成2–5份体检，验证可复制性

### 阶段3：公开招募合伙人（N人）

**前提**：
- 内圈合伙人试点跑顺，发现的坑已基本填平
- 形成标准化的培训大纲、话术手册、考核与结算规则

**关键动作**：
- 在朋友圈/公众号/社群发布「贷查查合伙人计划」
- 用在线文档 + 录屏 + 工具，降低培训与上手成本
- 建立合伙人分级机制（活跃度/质量），避免管理失控

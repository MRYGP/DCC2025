## 一、我们是什么？

> **不是贷款中介，是"贷款体检医生"**

**核心差异**：
- 传统中介：从"帮你贷款"出发，目标是促成交易
- 贷查查：从"帮你看清自己能贷什么"出发，目标是透明决策

**定位**：
- 我们不做"包下款"的承诺
- 我们不做"收中介费"的生意
- 我们做的是：**先体检，再决策**

---

## 二、我们解决什么问题？

### 2.1 企业端：信息黑箱 → 透明可视

**痛点**：
- 不知道能贷多少，心里没底
- 多家中介给不同方案，不知道谁靠谱
- 申请被拒了，不知道为什么

**解决方案**：
- 先做体检，看清楚自己的贷款体质
- 每个推荐都有明确依据，不是拍脑袋
- 知道"为什么能贷这么多""为什么利率是这个水平"

### 2.2 中介端：经验依赖 → 系统赋能

**痛点**：
- 服务质量看个人水平
- 新人上手慢，容易出错
- 每个客户都从头分析

**解决方案**：
- 标准化诊断，新手也能达到及格线
- 系统给方案，你做关系和服务
- 专业度可量化，客户更信任

### 2.3 银行端：被动接受 → 精准匹配

**痛点**：
- 收到大量不符合条件的申请
- 审批效率低，资源浪费

**解决方案**：
- 前置筛选，只推荐符合条件的客户
- 材料标准化，减少反复沟通
- 提高通过率，降低银行成本

---

## 三、我们的核心价值

### 3.1 将复杂金融决策转化为结构化数据

**以前**：
- 贷款能力是模糊的、依赖经验的
- 每个人说法不同，没有标准

**现在**：
- 用"贷款体质六宫格"量化评估
- 用"体检报告"可视化呈现
- 用"匹配算法"标准化推荐

### 3.2 将模糊的"贷款能力"转化为可视化的"体质等级"

**体质分级**：
- **A级**：优质客户，多产品可选，利率优惠
- **B级**：一般客户，部分产品可选，利率正常
- **C级**：风险客户，产品受限，需要改造

**价值**：
- 企业知道自己"在哪个等级"
- 知道"怎么提升等级"
- 知道"提升后能贷多少"

### 3.3 把"贷款这门手艺"变成一个系统

**以前**：
- 贷款是"手艺活"，依赖个人经验
- 老手和新手差距巨大
- 知识难以传承和复制

**现在**：
- 把经验变成规则
- 把规则变成算法
- 把算法变成系统

## 四、飞轮效应

```
企业信任系统 
  → 中介愿意使用 
    → 平台积累数据 
      → 算法持续优化 
        → 服务更精准 
          → 更多企业信任...
```

**具体表现**：
1. **企业端**：体检准确 → 信任系统 → 愿意付费 → 推荐给朋友
2. **中介端**：系统好用 → 提高效率 → 赚更多钱 → 推荐给同行
3. **平台端**：数据积累 → 算法优化 → 服务更准 → 吸引更多用户

**关键指标**：
- 体检准确率（企业反馈）
- 中介使用率（活跃度）
- 数据积累量（案例数）
- 算法优化频率（迭代速度）

## 五、差异化优势

### 5.1 可解释性强

**特点**：
- 每个推荐都有明确规则依据
- 不是"黑箱"算法，可以解释"为什么"
- 客户能看到完整的诊断逻辑

**价值**：
- 建立信任：客户知道"不是忽悠"
- 降低风险：每个决策都有依据
- 便于优化：知道哪里有问题

### 5.2 标准化诊断

**特点**：
- 不依赖个人经验
- 新手也能达到及格线
- 服务质量可量化

**价值**：
- 降低门槛：不是只有"老手"才能做
- 提高效率：不需要每个客户都从头分析
- 保证质量：有标准，有检查

### 5.3 数据沉淀

**特点**：
- 每次服务都在积累"行业知识"
- 数据越多，算法越准
- 形成正向循环

**价值**：
- 持续优化：越用越好用
- 形成壁垒：数据是核心竞争力
- 行业价值：成为"公共知识基础设施"

## 六、长期愿景

### 6.1 当前状态

中小企业贷款领域的知识是：
- **模糊的**：每个人说法不同，没有标准
- **依赖人情的**：认识谁比懂什么更重要
- **充满信息壁垒的**：银行不说、中介不透明

### 6.2 我们要做的

把这些知识变成：
- **结构化的**：可存储、可查询、可复用
- **可复用的**：新人也能用，不需要重新学习
- **可进化的**：越用越准，持续优化

### 6.3 终极目标

成为中小企业金融领域的"公共知识基础设施"

**不是**：
- ❌ 最大的贷款中介
- ❌ 最赚钱的金融平台
- ❌ 最知名的品牌

**而是**：
- ✅ 最懂贷款规则的"知识平台"
- ✅ 最透明的"体检系统"
- ✅ 最可复用的"公共基础设施"

## 八、诚实反思：哪些是真的，哪些还需验证

> **这一章是内部视角，不对外展示。**  
> 目的是避免"自我合理化"，保持对现实的清醒认知。

### 8.1 一个残酷的问题

**客户为什么来找你？**

| 我们说的 | 可能的真相 |
|---------|-----------|
| "想做贷款体检" | ❌ 没人会这么想 |
| "想了解自己贷款能力" | ❓ 可能，但不是主要动机 |
| "想贷款，不知道找谁" | ✅ 这才是真实需求 |
| "被其他中介坑过" | ✅ 这才是痛点 |

**真相**：客户要的是**"能贷到钱"**，不是"体检报告"。

---

### 8.2 系统的真实价值是什么？

**问**：如果没有贷查查系统，你还能做这门生意吗？

**诚实的答案**：能。靠经验、靠关系、靠对行业的理解，也能做。

**那系统的价值是什么？**
- ✅ **放大器**：让一个人能服务更多客户
- ✅ **标准化**：让新人也能达到及格线
- ✅ **可复制**：让合伙人能用
- ❌ **不是发动机**：发动机是松阳的经验和判断

---

### 8.3 价值主张验证状态

| 我们说的 | 验证状态 | 说明 |
|---------|---------|------|
| "诊断更准" | ❌ 未验证 | 没跑过实测，不知道准不准 |
| "产品更全" | ❌ 未验证 | 才20个产品，老中介可能比我们多 |
| "服务更透明" | ❓ 待验证 | 客户真的在意吗？还是只在意能不能贷到？ |
| "可解释性强" | ❓ 待验证 | 因为没有复杂算法，所以"可解释"——这是优势还是因为做不到？ |
| "飞轮效应" | ❌ 未验证 | 第一圈都没转起来，谈什么飞轮 |
| "公共知识基础设施" | ❌ 未验证 | 目前就是几个人在整理文档 |

---

### 8.4 哪些可能是真的？

| 点 | 为什么可能是真的 |
|----|-----------------|
| **"信息不对称是行业痛点"** | 这是客观事实，不是我们编的 |
| **"把经验变成系统有价值"** | 如果能做到，确实有价值——但前提是"能做到" |
| **"结构化数据比拍脑袋好"** | 逻辑上成立——但前提是数据要准 |

---

### 8.5 哪些可能是自嗨？

| 点 | 为什么可能是自嗨 |
|----|-----------------|
| **"贷款体检医生"** | 好听，但客户不是来"体检"的，是来"贷款"的 |
| **"飞轮效应"** | 商业计划书语言，现在连第一圈都没转 |
| **"公共知识基础设施"** | 愿景太宏大，目前只是几个文档 |

---

### 8.6 更诚实的定位版本

**版本A：对外版（包装过的）**
> "我们是贷款体检医生，帮企业做体质诊断，将复杂金融决策转化为结构化数据。"

**版本B：内部版（诚实的）**
> "我们是一个想把贷款中介这门生意做得更规范的团队。我们在尝试把老中介脑子里的经验整理成系统，这样新人也能用、客户也能看懂。目前还在验证阶段，不知道行不行得通。"

**版本C：务实版（对客户说的）**
> "我们帮中小企业搞清楚：你现在能贷多少、在哪贷、怎么准备材料。我们的优势是：比传统中介更透明，不忽悠你。"

---

### 8.7 当前最重要的事

**不是完善"价值主张文档"，而是**：

1. **用公司A跑一遍完整流程**
   - 诊断 → 推荐产品 → 申请 → 看结果
   - 推荐的产品批了吗？额度估对了吗？

2. **拿真实结果来验证/修正价值主张**
   - 批了：说明诊断逻辑有效
   - 没批：说明需要调整

3. **再来更新这份文档**
   - 基于真实验证的结果，而不是逻辑推演

---

### 8.8 验证清单

| 待验证项 | 验证方式 | 状态 |
|---------|---------|------|
| 诊断准不准 | 用公司A跑，看推荐产品能不能批 | ⏳ 待验证 |
| 额度估算准不准 | 看预估额度和实际批的差多少 | ⏳ 待验证 |
| 客户愿意为"体检"付费吗 | 问真实客户 | ⏳ 待验证 |
| 合伙人愿意用吗 | 找2-3个合伙人试用 | ⏳ 待验证 |
| 比老中介更准吗 | 同一个客户，对比推荐结果 | ⏳ 待验证 |

---

### 8.9 这一章的价值

> **避免"自嗨"的唯一方法，是不断问自己：这个验证过吗？**
>
> 李理铖的分析很好，但他分析的是"我们的文档"，不是"市场的反馈"。
>
> 基于文档的分析，只能证明"逻辑自洽"。
> 基于实测的结果，才能证明"真的有用"。

## 九、真实差异化：银行给不了的东西

### 9.1 四层价值体系

| 层次 | 价值 | 银行能给吗 | 我们能给吗 |
|------|------|-----------|-----------|
| **第1层** | 单银行额度预测 | ✅ 能 | ✅ 能 |
| **第2层** | 跨银行额度对比 | ❌ 不能 | ✅ 能 |
| **第3层** | 组合策略设计 | ❌ 不能 | ✅ 能 |
| **第4层** | 改造前后对比 | ❌ 不能 | ✅ 能 |

**银行只能做第1层，我们能做到第4层。这才是真正的差异化。**

### 9.2 组合策略示例

```
企业情况：年纳税8万，年开票300万，有房产估值400万

银行A只会说："你在我行能贷税贷80万"
银行B只会说："你在我行能贷发票贷60万"
银行C只会说："你在我行能贷抵押贷280万"

贷查查能给出的整体方案：

┌─────────────────────────────────────────────────────────────────┐
│  方案A（保守-低息优先）：                                         │
│  - 建行云税贷：80万（利率3.15%）                                 │
│  - 总额：80万，综合利率3.15%                                     │
│  - 适合：不急用钱，追求低成本                                    │
├─────────────────────────────────────────────────────────────────┤
│  方案B（平衡型）：                                                │
│  - 建行云税贷：80万（利率3.15%）                                 │
│  - 交行发票贷：50万（利率4.5%）                                  │
│  - 总额：130万，综合利率3.68%                                    │
│  - 适合：需要一定额度，又想控制成本                               │
├─────────────────────────────────────────────────────────────────┤
│  方案C（激进-额度优先）：                                         │
│  - 建行云税贷：80万（利率3.15%）                                 │
│  - 交行发票贷：50万（利率4.5%）                                  │
│  - 招行抵押贷：200万（利率2.98%）                                │
│  - 总额：330万，综合利率3.21%                                    │
│  - 适合：急需大额资金，有抵押物                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 9.3 改造前后对比示例

```
当前状态：年开票100万，年纳税2万
当前可贷：约30万（微众微业贷）

┌─────────────────────────────────────────────────────────────────┐
│  如果做开票提升（H业务）：                                        │
│  - 改造后：年开票500万，年纳税10万                               │
│  - 改造周期：6个月                                               │
│  - 改造成本：约X万                                               │
│  - 改造后可贷：约150万（税贷80万 + 发票贷70万）                   │
│  - 增加额度：120万                                               │
│  - ROI：增贷120万 vs 成本X万                                     │
├─────────────────────────────────────────────────────────────────┤
│  如果做高新认定：                                                 │
│  - 改造后：获得高新技术企业资质                                   │
│  - 改造周期：8-12个月                                            │
│  - 改造成本：约X万                                               │
│  - 改造后可贷：约200万（高新企业贷）                              │
│  - 增加额度：170万                                               │
│  - ROI：增贷170万 vs 成本X万                                     │
└─────────────────────────────────────────────────────────────────┘
```

### 9.4 为什么银行不能提供这些？

| 内容 | 银行为什么不能提供 |
|------|-------------------|
| 跨银行对比 | 银行只推自己的产品，不会告诉你去别家 |
| 组合策略 | 银行不会帮你设计"在我这贷一点，在别家贷一点" |
| 改造建议 | 相当于教客户"怎么包装"，监管红线 |
| 改造前后对比 | 暗示客户"做XX就能多贷"，银行不能说 |

### 9.5 公信力来源

**公信力 = 预测准确率**

| 如果 | 那么 |
|------|------|
| 贷查查说能贷120-150万，实际批了130万 | 公信力+1 |
| 贷查查说能贷120-150万，实际只批了60万 | 公信力-10 |

**积累机制**：
- 每个实测案例都记录"预测vs实际"
- 100个案例后，可以对外说"预测准确率XX%"
- 这比任何概念包装都有说服力

## 十、真实用户重新定义

### 10.1 用户分层

| 用户类型 | 他们的需求 | 用贷查查做什么 | 优先级 |
|---------|-----------|---------------|--------|
| 企业主（直接用） | 想知道能贷多少 | 基础体检 | P2 |
| **贷款中介** | 说服客户、展示专业度 | 跨银行对比 + 组合策略 | **P0** |
| **资质服务商** | 说服企业主做高新认定 | 改造前后对比 | **P0** |
| **带量服务商** | 说服企业主做开票提升 | 改造前后对比 | **P0** |

### 10.2 核心认知

**企业主不是主要用户，"中间服务商"才是。**

- 企业主是最终买单的人
- 但"中间服务商"才是真正使用贷查查的人
- 他们用贷查查来**说服企业主**

### 10.3 合规边界

| 后端类型 | 我们的态度 | 文档策略 |
|---------|-----------|---------|
| 资质认定（科小/高新） | ✅ 积极推 | 主要改造方案 |
| 开票提升（H业务） | ⚠️ 可做 | 强调"真实业务基础" |
| 纳税提升 | ✅ 可做 | 合规路径 |
| 灰色流水 | ❌ 不碰 | 文档中不提 |

## 十二、一句话总结（修正版）

> **贷查查的真正价值不是"体检"，而是"银行给不了的全局视角"：**
> - **跨银行对比**（银行只看自己）
> - **组合策略**（最优配比，不是单产品最大化）

> - **预测准确性**（公信力来自预测vs实际的吻合度）
>
> **用户不是企业主，是"中间服务商"——他们用贷查查说服企业主。**

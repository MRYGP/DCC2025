# 贷查查 · 产品定位与价值主张 v2.0

**版本**：v2.0  
**更新日期**：2025-12-26  
**更新内容**：整合战略反思，明确分层设计和商业闭环  
**用途**：产品定位、对外宣传、战略对齐

---

## 一、一句话定位

### 1.1 对外定位（公开）

> **"贷查查 = 你的贷款能力Dashboard"**  
> 银行政策在变，你的条件在变，你的贷款能力也在变。随时测一测，不错过任何机会。

### 1.2 内部定位（核心）

> **"贷查查是我们业务体系的前端入口 + 合伙人的专属武器"**  
> 不是开放的行业基础设施，而是有控制的漏斗系统。

---

## 二、贷查查的双重身份

### 2.1 对外：免费的贷款体检入口

**功能**：
- 企业主可以免费测
- 传播快、用户多
- 但只给"概要"，不给"详情"
- 想要详情 → 进入我们的漏斗

**目的**：
- 引流获客
- 建立品牌认知
- 让客户主动联系我们

### 2.2 对内：合伙人的专属武器

**功能**：
- 完整功能、改造方案、后端对接
- 合伙人用来获客、说服客户
- 客户进入我们的体系

**目的**：
- 让合伙人有独特价值
- 客户流向我们的后端服务
- 形成商业闭环

---

## 三、真实差异化：银行给不了的东西

### 3.1 四层价值体系

| 层次 | 价值 | 银行能给吗 | 我们能给吗 |
|------|------|-----------|-----------|
| **第1层** | 单银行额度预测 | ✅ 能 | ✅ 能 |
| **第2层** | 跨银行额度对比 | ❌ 不能 | ✅ 能 |
| **第3层** | 组合策略设计 | ❌ 不能 | ✅ 能 |
| **第4层** | 改造前后对比 | ❌ 不能 | ✅ 能 |

**银行只能做第1层，我们能做到第4层。这才是真正的差异化。**

### 3.2 组合策略示例

```
企业情况：年纳税8万，年开票300万，有房产估值400万

银行A只会说："你在我行能贷税贷80万"
银行B只会说："你在我行能贷发票贷60万"
银行C只会说："你在我行能贷抵押贷280万"

贷查查能给出的整体方案：

┌─────────────────────────────────────────────────────────────────┐
│  方案A（保守-低息优先）：                                         │
│  - 建行云税贷：80万（利率3.15%）                                 │
│  - 总额：80万，综合利率3.15%                                     │
│  - 适合：不急用钱，追求低成本                                    │
├─────────────────────────────────────────────────────────────────┤
│  方案B（平衡型）：                                                │
│  - 建行云税贷：80万（利率3.15%）                                 │
│  - 交行发票贷：50万（利率4.5%）                                  │
│  - 总额：130万，综合利率3.68%                                    │
│  - 适合：需要一定额度，又想控制成本                               │
├─────────────────────────────────────────────────────────────────┤
│  方案C（激进-额度优先）：                                         │
│  - 建行云税贷：80万（利率3.15%）                                 │
│  - 交行发票贷：50万（利率4.5%）                                  │
│  - 招行抵押贷：200万（利率2.98%）                                │
│  - 总额：330万，综合利率3.21%                                    │
│  - 适合：急需大额资金，有抵押物                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 3.3 改造前后对比示例

```
当前状态：年开票100万，年纳税2万
当前可贷：约30万（微众微业贷）

┌─────────────────────────────────────────────────────────────────┐
│  如果做开票提升（H业务）：                                        │
│  - 改造后：年开票500万，年纳税10万                               │
│  - 改造周期：6个月                                               │
│  - 改造成本：约X万                                               │
│  - 改造后可贷：约150万（税贷80万 + 发票贷70万）                   │
│  - 增加额度：120万                                               │
│  - ROI：增贷120万 vs 成本X万                                     │
├─────────────────────────────────────────────────────────────────┤
│  如果做高新认定：                                                 │
│  - 改造后：获得高新技术企业资质                                   │
│  - 改造周期：8-12个月                                            │
│  - 改造成本：约X万                                               │
│  - 改造后可贷：约200万（高新企业贷）                              │
│  - 增加额度：170万                                               │
│  - ROI：增贷170万 vs 成本X万                                     │
└─────────────────────────────────────────────────────────────────┘
```

### 3.4 为什么银行不能提供这些？

| 内容 | 银行为什么不能提供 |
|------|-------------------|
| 跨银行对比 | 银行只推自己的产品，不会告诉你去别家 |
| 组合策略 | 银行不会帮你设计"在我这贷一点，在别家贷一点" |
| 改造建议 | 相当于教客户"怎么包装"，监管红线 |
| 改造前后对比 | 暗示客户"做XX就能多贷"，银行不能说 |

## 四、分层开放设计

### 4.1 公开版（企业主/任何人）

**能看到**：
- 基础诊断结果
- 体质分级（A/B/C）
- 当前可贷额度预测
- 跨银行产品对比
- 组合策略建议
- 改造建议（概要）："如果提升XX，预计可贷XX万"

**看不到**：
- 改造方案详情（具体怎么做、成本、周期）
- 后端服务商对接

**关键入口**：
- 每份报告都有"想了解详情？联系我们的顾问"

### 4.2 合伙人版（专属）

**全部功能**：
- 公开版所有内容
- 改造方案完整详情（步骤、成本、周期、ROI）
- 直接对接我们的后端服务商
- 客户管理功能
- 历史记录对比

### 4.3 为什么这样设计？

| 设计 | 目的 |
|------|------|
| 公开版不给详情 | 想知道"怎么改造"必须联系我们 |
| 报告里有入口 | 即使别人用了，客户也可能来找我们 |
| 合伙人专属版 | 让合伙人有独特价值，客户进我们体系 |

## 五、用户分层

### 5.1 三类用户

| 用户类型 | 版本 | 使用频率 | 核心需求 | 流向 | 优先级 |
|---------|------|---------|---------|------|--------|
| **合伙人** | 专属版 | 高频 | 获客、说服客户 | 客户→我们体系 | P0 |
| **持续型企业主** | 公开版 | 中频 | 监控贷款能力变化 | 想详情→联系我们 | P0 |
| **一次性企业主** | 公开版 | 低频 | 了解能贷多少 | 可能转化 | P2 |
| **外部服务商** | 公开版 | - | 给客户诊断 | 我们不亏 | P2 |

### 5.2 为什么不怕外部服务商用？

| 他们用了 | 但是 |
|---------|------|
| 能给客户做基础诊断 | 没有详细改造方案 |
| 能告诉客户"可以改造" | 不知道具体怎么做 |
| 客户看到报告 | 报告里有我们的入口 |

## 六、商业闭环

### 6.1 完整流程图

```
企业主自己测（公开版）          合伙人给客户测（专属版）
      │                              │
      ↓                              ↓
   贷查查诊断系统（基础诊断+改造概要）
      │                              │
      │想了解详情                     │有完整方案
      ↓                              ↓
  【联系我们的顾问】←────────────────┘
      │
      ↓
   我们的客服/顾问
   - 企业主自己来的 → 分配给合伙人
   - 合伙人带来的 → 直接对接后端
      │
      ↓
   我们的后端服务商
   - 资质认定（科小/高新）
   - 带量服务（H业务）
   - 银行渠道对接
      │
      ↓
   成交 → 分润
```

### 6.2 关键控制点

| 控制点 | 设计 | 目的 |
|--------|------|------|
| 公开版不给详情 | 只说"可以"，不说"怎么" | 必须联系我们 |
| 报告里有入口 | 每份报告都有 | 客户可能自己来 |
| 后端绑定 | 只对接我们的服务商 | 不流失到外部 |

## 七、公信力来源

### 7.1 核心逻辑

> **公信力 = 预测准确率**  
> "贷查查说能贷120-150万，实际批了130万" → 这就是公信力

### 7.2 积累机制

- 每个实测案例都记录"预测vs实际"
- 100个案例后，可以对外说"预测准确率XX%"
- 这比任何概念包装都有说服力

### 7.3 验证清单

| 待验证项 | 验证方式 | 状态 |
|---------|---------|------|
| 单产品额度预测准确率 | 公司A实测 | ⏳ 待验证 |
| 组合策略可行性 | 多银行同时申请 | ⏳ 待验证 |
| 改造前后对比准确性 | 改造后再测 | ⏳ 待验证 |

## 八、诚实反思：哪些是真的，哪些还需验证

> **本章为内部视角，不对外展示。目的是避免"自我合理化"，保持清醒。**

### 8.1 残酷的问题

**客户为什么来找你？**

| 我们说的 | 可能的真相 |
|---------|-----------|
| "想做贷款体检" | ❌ 没人会这么想 |
| "想了解自己贷款能力" | ❓ 可能，但不是主要动机 |
| "想贷款，不知道找谁" | ✅ 这才是真实需求 |
| "被其他中介坑过" | ✅ 这才是痛点 |

**真相**：客户要的是**"能贷到钱"**，不是"体检报告"。

### 8.2 系统的真实价值

**问**：如果没有贷查查系统，你还能做这门生意吗？

**诚实答案**：能。靠经验、关系、行业理解也能做。

**那系统的价值是什么？**
- ✅ **放大器**：让一个人能服务更多客户
- ✅ **标准化**：让新人也能达到及格线
- ✅ **可复制**：让合伙人能用
- ❌ **不是发动机**：发动机是经验和判断

### 8.3 价值主张验证状态

| 我们说的 | 验证状态 | 说明 |
|---------|---------|------|
| "诊断更准" | ❌ 未验证 | 没跑过实测 |
| "产品更全" | ❌ 未验证 | 才20个产品 |
| "服务更透明" | ❓ 待验证 | 客户真的在意吗？ |
| "飞轮效应" | ❌ 未验证 | 第一圈都没转 |

### 8.4 三个版本的定位

**对外版（包装过的）**：
> "我们是贷款体检医生，帮企业做体质诊断。"

**内部版（诚实的）**：
> "我们在尝试把老中介的经验整理成系统，目前还在验证。"

**务实版（对客户说的）**：
> "我们帮你搞清楚能贷多少、在哪贷、怎么组合最划算。"

### 8.5 当前最重要的事

1. 用公司A跑完整流程，验证预测准确性
2. 积累"预测vs实际"案例
3. 验证组合策略可行性

## 十、一句话总结

> **贷查查不是开放的行业基础设施，而是有控制的漏斗系统：**
> - **公开版**：免费引流，给概要不给详情
> - **合伙人版**：专属武器，完整功能+后端对接
> - **商业闭环**：企业主→贷查查→联系我们→后端→分润
> - **真实差异化**：跨银行对比+组合策略+改造前后对比（银行给不了）
> - **公信力**：来自预测准确率，不是概念包装

# 岗位说明：贷查查技术实习生（技术员）

> 版本：v1.0  
> 更新日期：2025-12-09  
> 所属项目：贷查查 / DCC2025（企业贷款体质诊断与改造系统）  
> 角色定位：技术负责人级实习生 / Agent 架构实习生  
> 主要协作人：松阳（产品 Owner）、会计方向产品实习生

---

## 一、岗位定位与使命

### 1.1 一句话定位

> 把松阳脑子里的「资金体检 + 贷款体质医生」体系，  
> 做成一套可落地的 **诊断引擎 + 银行产品知识库 + 内部工作台**，  
> 让任何人/AI 都能像医生一样给出体检结论和产品建议。

### 1.2 不做什么

- 不负责整理 Excel、抄数据（这是会计实习生的工作）；  
- 不负责单纯写"业务文案"（但需要读懂文档，抽象成规则和 schema）；  
- 不负责销售/对外谈客户（但会参与技术演示和内部培训）。

---

## 二、核心工作主线（3 条）

> 所有工作都可以归到这 3 条主线，优先级按 1 > 2 > 3。

### 主线1：诊断引擎开发（优先级最高）

**目标**：把 `03_诊断工具/诊断逻辑说明.md` 里的逻辑，做成可运行的代码。

**具体任务**：
1. **问卷系统**：实现 15 道题的问卷（参考 `诊断问卷v1.0.md`）
   - 前端：表单渲染、数据收集、进度提示
   - 后端：数据验证、标准化处理

2. **产品匹配算法**：实现产品筛选逻辑（参考 `匹配算法说明.md`）
   - 读取银行产品库（JSON/YAML）
   - 逐个检查准入条件
   - 计算可贷额度
   - 排序输出

3. **报告生成**：根据匹配结果生成体检报告（参考 `体检报告模板.md`）
   - 组装报告数据结构
   - 生成诊断建议文案
   - 渲染 PDF/HTML

**技术栈建议**：
- 后端：FastAPI（Python）或 Express（Node.js）
- 前端：React/Vue + Tailwind CSS
- 数据存储：JSON 文件（初期）→ 数据库（后期）

---

### 主线2：知识库工具化（优先级第二）

**目标**：让银行产品库、改造方案库变成"可查询、可更新"的工具。

**具体任务**：
1. **产品库管理工具**：
   - 读取 `01_银行政策库/` 下的产品文档
   - 解析成结构化数据（JSON Schema）
   - 提供查询接口（按类型、按银行、按额度范围）

2. **方案库管理工具**：
   - 读取 `02_改造方案库/` 下的方案文档
   - 解析成结构化数据
   - 提供推荐接口（根据缺口类型推荐方案）

3. **知识库更新工具**：
   - 提供 Web 界面或 CLI 工具
   - 方便非技术人员更新产品/方案信息
   - 自动验证数据格式

---

### 主线3：内部工作台（优先级第三）

**目标**：给业务团队一个"看数据、管客户、跟踪进度"的工具。

**具体任务**：
1. **客户管理**：
   - 客户档案（基本信息、体检记录、方案推荐）
   - 跟进记录（沟通记录、方案执行进度）

2. **数据看板**：
   - 体检数量统计
   - 产品匹配率
   - 方案推荐率
   - 转化漏斗

3. **任务管理**：
   - 待办事项（验证清单、客户跟进）
   - 提醒通知（客户生日、方案到期）

---

## 三、工作方式与协作

### 3.1 文档驱动开发

**原则**：先有文档，再有代码。

**流程**：
1. 松阳/业务团队写需求文档（PRD、SOP、话术）
2. 你读文档，抽象成技术方案
3. 写代码实现
4. 测试验证
5. 更新文档（如有变更）

**关键文档位置**：
- 产品需求：`05_开发文档/PRD-贷查查v1.0.md`
- 诊断逻辑：`03_诊断工具/诊断逻辑说明.md`
- 数据库设计：`05_开发文档/数据库设计.md`
- API 设计：`05_开发文档/API接口设计.md`

---

### 3.2 与会计实习生的协作

**分工**：
- 会计实习生：整理银行产品数据、写业务文档、测试功能
- 你：把文档和数据变成代码

**协作方式**：
- 她整理好产品数据（Excel/文档）→ 你解析成代码可用的格式
- 她测试功能发现问题 → 你修复 bug
- 她提出新需求 → 你评估技术可行性

---

### 3.3 与松阳的协作

**沟通频率**：每天至少一次（微信/当面）

**沟通内容**：
- 当前任务进度
- 遇到的技术问题
- 需求理解确认
- 技术方案讨论

**产出物**：
- 代码提交（GitHub）
- 技术文档更新
- 问题记录（遇到问题及时记录，方便复盘）

## 六、常见问题

### Q1：我不会某个技术栈怎么办？

**A**：没关系，边学边做。遇到问题：
1. 先查文档/教程
2. 问 ChatGPT/Claude
3. 问松阳或技术朋友
4. 记录学习过程，形成知识沉淀

---

### Q2：需求不明确怎么办？

**A**：
1. 先读业务文档，理解业务逻辑
2. 列出不确定的问题
3. 找松阳确认
4. 确认后再开始写代码

---

### Q3：代码写不出来怎么办？

**A**：
1. 先写伪代码，理清逻辑
2. 拆解成小任务，逐个实现
3. 参考开源项目（GitHub）
4. 用 AI 辅助（GPT/Claude 写代码）
5. 实在不行，先做简化版，再迭代

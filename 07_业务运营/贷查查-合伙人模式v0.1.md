# 贷查查-合伙人模式 v0.1

版本：v0.1  
创建日期：2025-12-10  
适用范围：DCC2025 项目（贷查查）相关的获客、合伙人、返佣结算等业务设计

---

## 1. 一句话总结

> 用「贷查查 · 企业贷款体检」做 **免费入口产品**，  
> 把"能不能贷、能贷多少、怎么提升"讲清楚，  
> 后续产生的 **贷款、过桥、摆账/做量、典当、结构调整** 都交给后端合作方，  
> 我们只做：**诊断 + 分诊 + 转介**，吃的是 **后端返佣 / 咨询服务费**。

### 1.1 产品定位（面向助贷中介）

> **贷查查 = 助贷中介的"诊断与方案引擎"**
> 
> 帮你快速算清客户能贷多少、适合哪些产品，用标准化体检报告+方案 A/B/C，
> 提升专业度和成单率，避免纯靠经验"拍脑袋"。

**给助贷中介的 3 条价值**：

1. **客户一进来就有标准体检**
   - 用统一的 7 连问 / 诊断问卷采集信息，避免"漏问关键问题"。

2. **不用每单都重新算一遍**
   - 系统自动给出 A/B/C 方案和产品候选，
   - 中介只需要在几个可行方案里做选择和沟通。

3. **专业度和信任感更高**
   - 拿着"体检报告 + 方案说明"跟老板沟通，比单纯靠嘴说更有说服力；
   - 长期来看，更容易变成"客户的长期财务顾问"，而不是一次性的"贷款掮客"。

## 2. 整体模式：前端免费体检，后端多产品变现

### 2.1 前端：贷查查（免费体验）

统一入口产品名称：**「贷查查 · 企业贷款体检」**

企业提交信息后，系统 / 人工输出：

1. **当前体质等级**：A / B / C 级贷款体质  
2. **方案 A/B/C 概览**（示例）：

   - 方案 A：按当前体质，**立刻可尝试的银行 + 产品 + 预计额度区间 + 利率区间**

   - 方案 B：做一点轻度优化（例如换产品线、换银行、做简单票据配合），**可以提升多少额度/降低多少利率**

   - 方案 C：需要做"体质改造"（H 业务思路），**预计 X–Y 个月后有机会做到多少额度/什么结构**

体检 + 报告本身 **对企业免费**。

### 2.2 后端：多条业务线统一挂在「诊断结果」后面

体检结果出来之后，所有收费业务都归到后端，典型包括：

1. **直接银行贷款 / 授信**  

   - 税贷、票贷、流贷、房抵贷、信用贷等  

   - 收益来源：银行渠道返佣 / 咨询服务费

2. **票据 / 过桥 / 摆账 / 冲量 / 做存款**  

   - 为达成授信或提升额度，短期做「走量 / 冲存款 / 摆账」  

   - 收益来源：中介返佣、资金使用费分成

3. **典当 & 其它资金通道**  

   - 抵押物不足 / 时效很急时，推荐典当行或其它资金方  

   - 收益来源：合作方返佣

4. **体质改造类项目（H 业务）**  

   - 开票提升术、纳税提升术、资质认定术、法人优化术、资产增持术、征信修复术等  

   - 收益来源：项目咨询费 + 合作机构返佣

> 关键原则：  
> **前端一律"诊断 + 方案免费"，所有收费行为都放在后端合作环节完成。**

## 3. 客户视角的标准流程（企业老板怎么体验）

### Step 1：发起体检（入口统一）

- 渠道：合伙人转介绍 / 朋友圈 / 小程序 / 线下活动  

- 老板只需要做两件事：

  1. 填写或口述一份 **贷款体检问卷（7 连问 + 诊断问卷）**  

  2. 提供基本资料（票、税、流水、资产、征信等，可以先口径+截图，后补资料）

### Step 2：拿到《贷款体检报告》

报告不超过 2 页，包含 5 个模块：

1. 现状体质：A/B/C 级，当前贷款结构简要说明（总额、成本、集中度等）  

2. 方案 A：在**不做体质改造**的前提下，可尝试的产品线 + 预计额度/利率区间  

3. 方案 B：可以通过轻度调整（例如换产品线、做票据/存款配合）提升额度/降利率的空间  

4. 方案 C：如做中长期体质改造，可以期待的额度/结构变化（示例预测）  

5. 风险提醒：哪些做法不建议、哪些是高风险行为

> 报告结尾一定要有一句：  
> "后面的所有操作方案，**您可以自己找熟悉的机构做，也可以用我们推荐的合作方**，我们前端体检本身不收费。"

### Step 3：老板决定走哪条路

三种典型选择：

1. 只要方案 A：  

   - 只做"当前能贷多少"的对接服务，快速出结果。

2. 想要方案 B：  

   - 接受"为提升额度/降成本"做一点结构优化（贷款结构、票据、存款、抵押等）。

3. 想要方案 C：  

   - 接受"体质改造"（H 业务类），愿意投入时间和费用，换更高的额度和更好的结构。

### Step 4：后端协同（贷款 / 过桥 / 典当 / 做量）

- 根据选择的方案，匹配对应的 **后端合作方**：  

  - 银行渠道 / 小微业务团队  

  - 票据公司 / 资金中介 / 典当行  

  - 体质改造服务机构（财税、资质、资本运作等）

- 企业与合作方直接签署服务协议 / 资金协议，  

  我们只参与：方案设计 + 进度跟踪 + 效果复盘。

### Step 5：结案 & 复盘

- 把关键数据沉淀进 DCC2025：

  - 企业体质档案  

  - 实际获批额度/利率  

  - 真实执行的"手术包"与花费  

  - 前后对比（ROI / 风险）

- 用于：

  - 优化诊断模型  

  - 优化方案库  

  - 给合伙人更多"有图有真相"的案例

## 4. 合伙人视角：只做 4 件事

合伙人定位：**"前端医生"+"客户关系维护者"**，而不是传统意义上的"贷款中介跑腿仔"。

### 合伙人要做的 4 步

1. 找到合适的企业老板

   - 有贷款 / 资金需求  

   - 或者：已有贷款，但利率偏高 / 额度不够 / 担保压力大

2. 完成一份合格的「贷款体检」

   - 使用：**贷款体检 7 连问 + 诊断问卷（中介版）**  

   - 能问清楚：贷款现状、票税流水、资产抵押、征信、痛点和目标

3. 把完整资料交到中台（我们）

   - 提交：体检问卷记录 + 基本票税流水资料（可先用截图/说明，后期补齐）  

   - 在系统里形成一条【项目编号】。

4. 把体检结果带回老板

   - 拿到《贷款体检报告》  

   - 和老板一起看：现在状况 & 方案 A/B/C  

   - 如果老板愿意走后端方案，协助建立三方沟通群（企业 + 我们 + 后端合作方）。

### 合伙人**不需要**做的事

- 不需要自己做复杂方案设计；  

- 不需要自己去找 N 家银行比价；  

- 不签任何资金合同，不代企业签字；  

- 不承诺"包下款""100%能做"这类高风险话术。

## 5. 返佣 & 分成（框架版）

> 此处为模式设计草案，所有比例仅做 **区间示例**，具体数值后续在《业务员运营与招募手册》里定稿。

### 5.1 收入来源

1. 合作方返佣  

   - 银行 / 渠道方 / 中介公司 / 典当行 / 服务机构 按项目回款比例支付服务费或返佣。

2. 咨询服务费（视情况）  

   - 针对复杂"体质改造项目"，可单独向企业收取体检+方案设计服务费。

### 5.2 分成逻辑（示意）

以"项目净佣金"为基数（扣除必要税费/硬成本之后）：

1. **轻服务单**（只对接银行 / 简单贷款方案）

   - 合伙人：约 30%  

   - 公司中台：约 70%

2. **中度服务单**（涉及多方协调、票据/过桥/做量组合等）

   - 合伙人：约 30–40%  

   - 公司中台：约 60–70%

3. **重服务单 / 体质改造项目**

   - 合伙人：约 20–30%  

   - 公司中台：约 70–80%

> 真实执行时，按项目类型 + 合伙人贡献度 + 是否有持续维护，  
> 在上述区间内打点，避免写死单一比例。

### 5.3 结算节奏（建议）

1. 小单（轻服务单）：按项目落地后 **单笔结算**；  

2. 大单 / 重服务单：可按 **阶段里程碑** 结算（如首款 / 中期 / 尾款）；  

3. 统一由中台出一份《项目结算单》，明确：

   - 项目编号  

   - 总返佣 / 咨询费  

   - 合伙人比例与金额  

   - 预计打款时间

## 6. 风险控制与对外话术边界

1. 对外绝不使用：

   - "包下款""100%能做""不成不收费但我们先垫钱" 这类表述；

2. 所有"预测额度 / 利率 / 时间"，统称为：

   - "预计""区间""在当前政策和您提供信息不变前提下"；

3. 报告中必须出现一句：

   - "以上预测仅为内部测算，**最终审批结果以银行及合作机构为准**。"

4. 合伙人只负责"体检 + 反馈 + 协调沟通"，  

   不代表任何资金方做法律承诺。

## 7. 落地分阶段规划（与 DCC2025 项目节奏对齐）

### Phase 1：创始团队自用（当前阶段）

- 仅内部使用：  

  - 由项目组自己用贷查查 + 7 连问，  

  - 针对自有企业和熟人企业，完成 **30–50 份体检**；  

  - 同时跑出若干真实案例（贷款 / 过桥 / 典当 / 做量 / 体质改造）。

- 产出：

  - 案例库首批内容（04_案例库/）  

  - 体检报告模板 & 话术  

  - 真实单子的"收入结构 + 成本结构"统计。

### Phase 2：内圈合伙人试点（3–5 人）

- 对象：熟悉的代账、保险、工程、贸易等圈层朋友；  

- 给他们 **简化版 SOP + 合伙人分成区间**；  

- 目标：评估

  - 他们能不能稳定交单？  

  - 合作体验如何？  

  - 哪些环节要改流程 / 话术。

### Phase 3：公开招募 N 个兼职合伙人（未来阶段）

- 条件：  

  - DCC2025 的诊断工具、案例库、合伙人 SOP 基本稳定；  

  - 结算规则、培训流程已跑通；

- 渠道：  

  - 朋友圈、公众号、社群等发布「贷查查合伙人计划」；  

  - 提供标准化的培训材料和考核机制。

> 本文档为 v0.1 版本，重点是把 **"用贷查查拓客 + 后端变现 + 合伙人协同"** 的整体骨架搭起来，  
> 具体分成比例、结算细节、合同条款，后续在《业务员运营与招募手册》《合作方协议模板》中逐步完善。

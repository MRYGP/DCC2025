# 📋 Cursor指令模板：新建银行产品

**用途**：标准化创建银行产品文档的指令格式  
**适用场景**：需要添加新的银行产品到知识库时

---

## 指令格式

```markdown
## 📋 Cursor执行任务

### 任务：创建银行产品文档

**目标**：在 `01_银行政策库/[产品类型]/` 目录下创建新的银行产品文档

**背景**：[说明为什么需要添加这个产品，例如：客户咨询、市场调研发现等]

---

### 文件操作

**操作类型**：创建新文件

**文件路径**：`01_银行政策库/[产品类型]/[银行名]-[产品名].md`

**内容要求**：

1. **标题格式**：`# [银行名] - [产品名]`

2. **基本信息**（必填）：
   - 产品名称
   - 银行名称
   - 产品类型（发票贷/税贷/高新企业贷/抵押贷/特殊人群贷/经营快贷）
   - 最高额度
   - 利率范围
   - 放款速度
   - 更新日期

3. **核心内容**（必填）：
   - 一、产品概述
   - 二、准入条件（详细列出所有条件）
   - 三、额度算法（如果有公式，详细说明）
   - 四、所需材料（清单）
   - 五、申请流程（步骤）
   - 六、备注（优势、劣势、适用场景、注意事项）

4. **数据来源**：标注信息来源（银行官网、公开资料、Web搜索等）

---

### 参考模板

参考现有产品文档的格式，例如：
- `01_银行政策库/税贷/农业银行-纳税e贷.md`
- `01_银行政策库/税贷/民生银行-民生税贷.md`

---

### Git操作

**Commit信息**：`add: 新增[银行名]-[产品名]产品文档`

**注意**：执行 git add + git commit，**不要执行 git push**（等待用户确认）

---

### 完成标准

- [ ] 文件已创建在正确位置
- [ ] 内容完整，符合模板要求
- [ ] 已更新产品总表（如有需要）
- [ ] 已提交到Git（未推送）
```

## 注意事项

1. **文件命名规范**：`[银行名]-[产品名].md`，使用中文短横线
2. **目录分类**：根据产品类型放入对应文件夹
3. **数据准确性**：确保所有数据真实可靠，标注数据来源
4. **格式统一**：参考现有产品文档的格式和结构
5. **更新总表**：创建新文档后，记得更新 `01_银行政策库/银行政策库-产品总表.md`

# 📋 Cursor指令模板：更新进度

**用途**：标准化更新项目进度追踪器的指令格式  
**适用场景**：完成某个任务后，需要更新项目进度追踪器时

---

## 指令格式

```markdown
## 📋 Cursor执行任务

### 任务：更新项目进度追踪器

**目标**：更新 `00_核心文档/项目进度追踪器-更新.md` 中的任务状态

**背景**：[说明完成了什么任务，例如：完成了方案XX的创建、修复了编号冲突等]

---

### 文件操作

**操作类型**：修改现有文件

**文件路径**：`00_核心文档/项目进度追踪器-更新.md`

**更新内容**：

1. **更新任务状态**：
   - 将对应任务的"状态"从 `⏳` 改为 `✅`
   - 更新"完成日期"为当前日期
   - 确认"文件位置"正确

2. **更新模块进度**：
   - 更新模块标题，例如：`模块2：改造方案扩展（7/7）✅`
   - 更新整体进度百分比

3. **更新总体进度**：
   - 更新Phase 1总体进度
   - 更新整体进度百分比

---

### 更新示例

**更新前**：
```markdown
| P2-6：方案04-征信修复术 | ⏳    | 待定     | 方案04-征信修复术.md          |
```

**更新后**：
```markdown
| P2-6：方案06-征信修复术 | ✅    | 2025-12-07 | 方案06-征信修复术.md          |
```

---

### Git操作

**Commit信息**：`update: 更新项目进度追踪器 - [任务描述]`

**注意**：执行 git add + git commit，**不要执行 git push**（等待用户确认）

---

### 完成标准

- [ ] 任务状态已更新为 ✅
- [ ] 完成日期已填写
- [ ] 模块进度已更新
- [ ] 总体进度已更新
- [ ] 已提交到Git（未推送）
```

---

---

### 文件操作

**文件路径**：`00_核心文档/项目进度追踪器-更新.md`

**更新内容**：

找到任务"P2-13：方案01-开票提升术"，更新为：

---

## 注意事项

1. **日期格式**：使用 `YYYY-MM-DD` 格式，例如：`2024-12-07`
2. **状态符号**：
   - `✅` 已完成
   - `⏳` 进行中/待定
   - `❌` 已取消
3. **进度计算**：确保模块进度和总体进度计算正确
4. **文件位置**：确保文件路径正确，与实际文件位置一致
5. **一致性**：确保进度追踪器中的信息与实际文件状态一致

## 常用更新场景

### 场景1：完成单个任务
- 更新该任务的状态、完成日期
- 更新对应模块的进度

### 场景2：完成整个模块
- 更新模块内所有任务的状态
- 更新模块标题为 `✅ 100%`
- 更新总体进度

### 场景3：修复问题
- 更新问题修复相关的任务状态
- 在备注中说明修复内容

# 📋 Cursor指令模板：新建改造方案

**用途**：标准化创建改造方案文档的指令格式  
**适用场景**：需要添加新的改造方案到知识库时

---

## 指令格式

```markdown
## 📋 Cursor执行任务

### 任务：创建改造方案文档

**目标**：在 `02_改造方案库/` 目录下创建新的改造方案文档

**背景**：[说明为什么需要添加这个方案，例如：新业务、客户需求等]

---

### 文件操作

**操作类型**：创建新文件

**文件路径**：`02_改造方案库/方案XX-[方案名称].md`

**编号规则**：
- 方案编号从01开始，依次递增
- 当前最新编号：方案11（流水优化术）
- 新方案编号：方案12

**内容要求**：

1. **标题格式**：`# 方案XX：[方案名称]`

2. **基本信息**（必填）：
   - 更新日期
   - 关联业务（H业务/G业务/外部服务）
   - 方案分类

3. **核心内容**（必填，参考方案00-模板.md）：
   - 一、方案概述（适用场景、改造目标、核心逻辑）
   - 二、执行步骤（Step 1、2、3...）
   - 三、成本分析（表格+说明）
   - 四、周期规划（表格+最快/最慢情况）
   - 五、效果预测（改造前后对比+ROI计算）
   - 六、风险提示（至少2个风险）
   - 七、成功案例（可选）
   - 八、话术要点（客户痛点挖掘、方案介绍、异议处理）
   - 九、操作清单（前期准备、执行期、后期跟进）

---

### 参考模板

参考以下文件：
- `02_改造方案库/方案00-模板.md`（标准模板）
- `02_改造方案库/方案01-开票提升术.md`（完整示例）

---

### 更新方案总表

创建新方案后，需要更新 `02_改造方案库/方案总表.md`：
- 在方案目录表格中添加新方案
- 更新方案数量统计

---

### Git操作

**Commit信息**：`add: 新增方案XX-[方案名称]`

**注意**：执行 git add + git commit，**不要执行 git push**（等待用户确认）

---

### 完成标准

- [ ] 文件已创建在正确位置
- [ ] 编号正确（不冲突）
- [ ] 内容完整，符合模板要求
- [ ] 已更新方案总表
- [ ] 已提交到Git（未推送）
```

---

## 注意事项

1. **编号规则**：确保编号不冲突，按顺序递增
2. **文件命名**：`方案XX-[方案名称].md`，使用中文短横线
3. **内容完整性**：必须包含模板中的所有9个章节
4. **格式统一**：参考方案00-模板.md的格式
5. **更新总表**：创建新方案后，必须更新方案总表

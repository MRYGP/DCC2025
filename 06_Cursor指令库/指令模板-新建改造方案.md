# 📋 Cursor指令模板：新建改造方案

**用途**：标准化创建改造方案文档的Cursor指令格式  
**适用场景**：当需要在知识库中新增改造方案时使用

---

## 指令格式

```markdown
## 📋 Cursor执行任务

### 任务：新建改造方案 - 方案XX-[方案名称]

**目标**：在 `02_改造方案库/` 目录下创建新的改造方案文档

**背景**：[说明为什么要添加这个方案，例如：客户需求、市场gap等]

---

### 文件操作

**操作类型**：创建新文件

**文件路径**：`02_改造方案库/方案XX-[方案名称].md`

**编号说明**：
- 查看 `02_改造方案库/方案总表.md` 确认下一个可用编号
- 当前已有：方案01-11
- 新方案编号：方案12（示例）

**方案分类**：
- 开票提升
- 纳税提升
- 资质认定
- 法人优化
- 资产增持
- 征信修复
- 抵押优化
- 账期优化
- 多头授信
- 贷款置换
- 流水优化
- [其他新分类]

**文件内容**：

[参考 `02_改造方案库/方案00-模板.md`，按照标准格式填写]

主要章节：
1. 方案概述（适用场景、改造目标、核心逻辑）
2. 执行步骤（Step 1-4，每步含动作、周期、成本、产出）
3. 成本分析（费用明细表、成本说明）
4. 周期规划（阶段表、最快/最慢情况）
5. 效果预测（前后对比表、ROI计算）
6. 风险提示（至少3个风险）
7. 成功案例（至少1个完整案例）
8. 话术要点（痛点挖掘、方案介绍、异议处理）
9. 操作清单（前期准备、执行期、后期跟进）

---

### 同步更新

**1. 更新方案总表**：
在 `02_改造方案库/方案总表.md` 的"方案目录"表格中添加：

```markdown
| XX | [方案名称] | [方案XX-[方案名称].md](./方案XX-[方案名称].md) | [方案分类] | 约X万 | X-X个月 | ⭐⭐⭐⭐ |
```

**2. 更新方案对比总表**（如适用）：
如果新方案与现有方案类似，在方案总表的"方案对比总表"中添加列。

**3. 更新项目进度追踪器**：
在 `00_核心文档/项目进度追踪器.md` 中标记该方案为已完成。

**4. 更新README**（如适用）：
如果是重要方案，在README的"改造方案类型"表格中添加。

---

### Git操作

**Commit信息**：`add: 新增改造方案 - 方案XX-[方案名称]`

**注意**：执行 git add + git commit，**不要执行 git push**（等待用户确认）

## 注意事项

### 编号管理
1. **查看方案总表**：先确认当前最大编号
2. **检查冲突**：确保新编号没有被占用
3. **连续编号**：建议使用连续编号（但不强制）

### 内容要求
1. **数据真实**：成本、周期基于实际案例或合理估算
2. **合规表述**：避免"保证""承诺""100%成功"等表述
3. **案例完整**：至少包含1个真实或虚拟的完整案例
4. **话术实用**：提供可直接使用的客户沟通话术

### 质量检查
1. **逻辑自洽**：方案逻辑清晰，执行步骤可行
2. **数据一致**：成本、周期、ROI数据前后一致
3. **格式统一**：遵循模板格式，保持与其他方案一致
4. **风险说明**：充分提示风险，不误导客户

## 常见问题

### Q1：如何确定方案编号？
A：
1. 打开方案总表，查看当前最大编号
2. 使用下一个编号（如当前最大是11，则用12）
3. 确保编号连续（可选）

### Q2：如果方案很简单，能不能简化？
A：
- 不建议。保持格式统一更重要
- 简单方案可以在某些章节写得简洁些
- 但9个章节都要保留

### Q3：ROI怎么计算？
A：
- 方式1：(预计新增额度 / 投入成本) × 100%
- 方式2：(预计节省利息 × N年 / 投入成本) × 100%
- 选择更合理的方式，或两个都列出

### Q4：没有真实案例怎么办？
A：
- 可以虚拟一个合理的案例
- 标注"虚拟案例"或"典型场景"
- 数据要合理，符合方案逻辑
